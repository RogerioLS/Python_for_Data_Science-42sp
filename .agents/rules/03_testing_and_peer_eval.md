# 03 — Testing and Peer Evaluation

Tests are defense ammunition.

## Minimum checks

Run or create:

- subject tester;
- import-only check;
- one valid edge case;
- one invalid input case;
- `python -m py_compile`;
- `flake8` when available.

## Peer defense readiness

Be able to explain:

- why each import is allowed;
- how invalid input is handled;
- why shapes/outputs match the subject;
- why no future exercise was mixed in;
- why the implementation is not overbuilt.

## Final report

Always report commands run and what remains untested.
