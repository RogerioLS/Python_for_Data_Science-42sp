# 00 — Global Rules

These rules apply to every task.

## Prime directive

Pass the subject first. Improve style second. Be clever never, unless the subject asked for cleverness.

## Always

- Read the relevant subject section before editing.
- Respect exact folder names and file names.
- Use Python 3.10-compatible code.
- Use explicit imports.
- Keep code import-safe.
- Use `main()` for executable scripts.
- Add docstrings to required functions/classes/methods.
- Handle expected errors.
- Keep changes small and scoped.

## Never

- Do not use global variables.
- Do not use `from x import *`.
- Do not add unrequested frameworks.
- Do not change expected output casually.
- Do not edit unrelated exercises.
- Do not claim tests passed without command output.
