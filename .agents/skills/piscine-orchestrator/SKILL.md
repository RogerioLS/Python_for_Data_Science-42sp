# Piscine Orchestrator

## Mission

Coordinate the Piscine without mixing modules or inventing requirements.

## Procedure

1. Identify target module and exercise.
2. Confirm subject source and required files.
3. Route to module-specific skill.
4. Produce a checklist before implementation.
5. Ensure validation evidence exists before calling work complete.

## Output

Return:

- module/exercise;
- required files;
- allowed imports/functions;
- implementation risks;
- validation commands;
- next step.
