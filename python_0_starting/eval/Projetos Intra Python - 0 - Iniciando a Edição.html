Repositório Git

    

    





    

    


    

    

    Introdução
    

    
- Mantenha-se educado, cortês, respeitoso e construtivo 
    durante todo o processo de avaliação. O bem-estar da comunidade 
    depende disso. 

    - Identifique-se com a pessoa (ou o grupo) avaliado(a) em relação às eventuais 
    disfunções do trabalho. Dedique tempo para discutir 
    e debater os problemas que você identificou. 

    - Considere que pode haver diferenças na forma como seus 
    colegas interpretaram as instruções do projeto e o 
    escopo de suas funcionalidades. Mantenha sempre a mente aberta e avalie 
    o trabalho da forma mais honesta possível. A pedagogia só é válida 
    se a avaliação por pares for conduzida com seriedade.
    

    

    

    

    Diretrizes
    

    
- Avalie apenas os trabalhos que estiverem no 
    repositório Git do aluno ou do grupo. 

    - Verifique novamente se o repositório Git pertence ao aluno 
    ou ao grupo. Certifique-se de que o trabalho seja referente ao projeto em questão 
    e verifique também se o comando "git clone" foi usado em uma pasta vazia. 

    - Verifique cuidadosamente se nenhum aliase malicioso foi usado para enganá-lo 
    e levá-lo a avaliar algo diferente do conteúdo do 
    repositório oficial. 

    - Para evitar surpresas, verifique cuidadosamente se tanto o 
    aluno avaliador quanto o avaliado revisaram os possíveis scripts usados 
    ​​para facilitar a avaliação. 

    - Se o aluno avaliador ainda não tiver concluído o 
    projeto em questão, é obrigatório que ele leia 
    todo o conteúdo do projeto antes de iniciar a defesa. 

    - Use os indicadores disponíveis nesta escala para sinalizar um repositório vazio, 
    um programa que não funciona, fraude, etc. 
    Nesses casos, a avaliação é encerrada e a nota final é 0, 
    ou -42 em caso de fraude. No entanto, exceto em casos de fraude, você 
    é encorajado a continuar discutindo seu trabalho, mesmo que ele esteja em 
    andamento, a fim de identificar quaisquer problemas que possam ter causado a 
    falha do projeto e evitar repetir o mesmo erro no futuro. 

    - Lembre-se de que, durante a defesa, não é permitida nenhuma 
    interrupção inesperada, prematura ou descontrolada do programa; caso contrário, a 
    nota final será 0 para o exercício e a avaliação continuará. 

    - Você nunca deve editar nenhum arquivo, exceto o arquivo de configuração, se 
    houver. Se precisar editar um arquivo, explique o motivo 
    ao aluno avaliado e certifique-se de que ambos concordem. 

    - As importações de bibliotecas devem ser explícitas; por exemplo, importar "from pandas import *" 
      não é permitido; nesse caso, você deve atribuir 0 ao exercício e continuar a avaliação. 

    - Seus exercícios serão avaliados por outros alunos; 
    certifique-se de que os nomes de variáveis ​​e funções sejam apropriados e respeitosos.
