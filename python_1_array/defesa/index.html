ex00: Give my BMI

File to turn in: give_bmi.py

COMPLETE

Objective & Solution Summary

Calculate the Body Mass Index (BMI) from lists of heights (meters) and weights (kilograms), and filter them against an integer limit threshold.


Implemented vectorized calculation using NumPy to perform division and exponentiation efficiently on full lists instead of looping manually.

# BMI Formula used in numpy
h_arr = np.array(height, dtype=float)
w_arr = np.array(weight, dtype=float)
bmi_arr = w_arr / (h_arr ** 2)

Critical Validation Checks

  • List Sizes: Validated that len(height) == len(weight), raising ValueError if they differ.
  • Zero/Negative Values: Height or weight $\le 0$ raises ValueError to prevent impossible BMIs and division by zero.
  • Type Enforcement: All items in lists must be int or float, raising TypeError otherwise.

How to Defend (Why & Q&A)

Q: Why use NumPy arrays instead of a standard loop?

"NumPy enables vectorization (vectorized operations). Pure Python loops are slow because Python has to check types on every single iteration. NumPy compiles the loop in C, allocating contiguous blocks of memory, executing math operations simultaneously across the array."

Q: Why check types using `type(x) not in (int, float)` instead of `isinstance`?

"In Python, bool is a subclass of int (i.e., isinstance(True, int) evaluates to True). If we used isinstance, boolean values like True or False would bypass the check and behave like 1 or 0, corrupting the BMI formula. Using type() guarantees strict type checks."

ex01: 2D Array

File to turn in: array2D.py

COMPLETE

Objective & Solution Summary

Inspect the shape of a 2D array, slice it row-wise using Python native slicing, print the shapes, and return the sliced list.


Leverages NumPy to fetch the shape tuple effortlessly, keeping the actual slice native to Python.

# Retrieve shape using NumPy
arr = np.array(family)
print(f"My shape is : {arr.shape}")
sliced = family[start:end]
print(f"My new shape is : {np.array(sliced).shape}")

Critical Validation Checks

  • Type Verification: Validated that family is a list, and that start and end are integers (preventing booleans).
  • Nested Type Verification: Verified that every row in the outer list is a list itself.
  • Inconsistent Row Sizes: Checked that all rows share the same length, raising a ValueError if they differ.

How to Defend (Why & Q&A)

Q: How does Python native slicing differ from NumPy slicing?

"Native Python list slicing on a 2D list only cuts rows (first dimension), because lists are 1D containers of other lists. NumPy slicing allows multi-dimensional index slicing (e.g., `arr[start:end, col_start:col_end]`), which is faster and allows sub-matrix slicing."

Q: What happens if rows have different sizes?

"If rows have different lengths, the structure is not a proper 2D array. When converted to NumPy, it becomes a 1D array of `object` dtype (shape `(N,)`), instead of a `(N, M)` matrix. That's why we explicitly validate row sizes and raise a `ValueError`."

ex02: Load my image

File to turn in: load_image.py

COMPLETE

Objective & Solution Summary

Load an image file (e.g. JPG or JPEG), print its dimensions (shape), and return its RGB pixel contents in a NumPy array format.


Uses Pillow (PIL) for image handling, explicitly converting the image format to RGB, and then converting the image into a NumPy array.

# Image loading logic
with Image.open(path) as img:
    img_rgb = img.convert("RGB")
    img_array = np.array(img_rgb)
    print(f"The shape of image is: {img_array.shape}")
    return img_array

Critical Validation Checks

  • Missing File: Catches FileNotFoundError if the target file does not exist.
  • Corrupted or Bad Format: Catches PIL.UnidentifiedImageError if the file exists but isn't an image.
  • Wrong Type: Validates path type is string, catching TypeError otherwise.

How to Defend (Why & Q&A)

Q: Why is it important to explicitly convert the image using `img.convert("RGB")`?

"Images can come in various modes, such as Grayscale (L), RGBA (with transparency channel), or Palette-based (P). Explicitly converting using convert("RGB") guarantees that we always receive a uniform 3-channel array (Red, Green, Blue) of shape (Height, Width, 3), preventing dimensional mismatches in downstream tasks."

Q: How are errors handled in `ft_load` to prevent the program from crashing?

"We wrap the Pillow loading function in a try-except block, catching specific exceptions like FileNotFoundError, UnidentifiedImageError, and generic Exception. If an error is caught, we print a descriptive message and return None, preventing unhandled exception crashes."

ex03: Zoom on me

Files to turn in: load_image.py, zoom.py

COMPLETE

Objective & Solution Summary

Load an image file, display its dimensions and content, slice a 400x400 area with a single color channel (grayscale) and display it with X and Y axis coordinate scales.


Uses NumPy slicing to crop the image array, and Matplotlib to draw it using a grayscale colormap with native axis scale ticks.

# Slice to 400x400 with 1 color channel
sliced = img[100:500, 380:780, 0:1]
print(f"New shape after slicing: {sliced.shape} or {sliced.shape[:2]}")

Critical Validation Checks

  • Grayscale Colormap: Used cmap="gray" inside plt.imshow() to display the single channel correctly.
  • Axis Scales: Kept axes visible so coordinate scales (0 to 400) appear on both X and Y dimensions.
  • Exceptions & Safety: Wrapped execution in a try-except block to catch FileNotFoundError, ImportError, or Matplotlib rendering failures gracefully.

How to Defend (Why & Q&A)

Q: How does slicing `img[100:500, 380:780, 0:1]` isolate the raccoon and shape it?

"Slicing specifies `[start_y:end_y, start_x:end_x, start_channel:end_channel]`. This selects a height of 400 pixels (from index 100 to 499), a width of 400 pixels (from index 380 to 779), and keeps only the first channel (Red channel, index 0). This results in a shape of `(400, 400, 1)`."

Q: Why does the print statement show `(400, 400, 1) or (400, 400)`?

"The sliced array has 3 dimensions `(400, 400, 1)` because of the slice range `0:1` on the channel dimension. However, `sliced.shape[:2]` drops the channel dimension, giving `(400, 400)`, which represents a 2D image matrix."

ex04: Rotate me

Files to turn in: load_image.py, rotate.py

COMPLETE

Objective & Solution Summary

Crop a 400x400 area from the loaded image with 1 color channel (grayscale) and transpose it manually without using any helper libraries for rotation/transposition.


Implemented manual transposition by instantiating a new NumPy array of swapped dimensions and populating it using nested for loops.

# Manual transposition loop
for i in range(height):
    for j in range(width):
        transposed[j, i] = sliced[i, j, 0]

Critical Validation Checks

  • Manual Transposition: Verified that no NumPy transposition shortcuts like .T, np.transpose(), or np.rot90() are used.
  • Grayscale Color Channel: Isolated single channel 0` (Red channel) and printed shapes.
  • Axis Scales: Matplotlib window displays coordinates for both X and Y dimensions (0 to 400).

How to Defend (Why & Q&A)

Q: What is the math behind a matrix transpose?

"Transposing swaps the rows and columns of a matrix. The element at index `(i, j)` in the original matrix becomes the element at index `(j, i)` in the transposed matrix. Mathematically: $T[j, i] = A[i, j]$."

Q: How did you implement this transpose manually?

"We initialize a destination array of shape `(width, height)`. We then use nested loops to iterate through all rows `i` and columns `j` of the sliced image, assigning `transposed[j, i] = sliced[i, j, 0]`."
