ex00: Give my BMI

  • Arquivos de entrega: give_bmi.py
  • Objetivo: Calcular o IMC ($peso / altura^2$) e verificar se os elementos da lista excedem um limite especificado.
P: Por que usar arrays do NumPy em vez de loops padrão do Python?
O NumPy permite **operações vetorizadas** (vectorized operations). Loops normais do Python são lentos porque precisam verificar dinamicamente o tipo das variáveis a cada iteração. O NumPy executa as operações matemáticas de forma concorrente em blocos contíguos de memória utilizando código C pré-compilado, tornando-o extremamente eficiente.
P: Por que validar os tipos na lista com `type(x) not in (int, float)` em vez de `isinstance`?
No Python, bool é uma subclasse de int (portanto isinstance(True, int) avalia para True). O uso de isinstance aceitaria incorretamente booleanos como números válidos nas listas, corrompendo o cálculo do IMC. O uso de type() direto previne essa falha.
P: Como a divisão por zero é evitada?
Nós validamos explicitamente se todos os elementos nas listas de altura e peso são estritamente maiores do que zero ($> 0$), lançando um ValueError antes que qualquer cálculo matemático ocorra.

[22.507863455018317, 29.0359168241966] <class 'list'>
[False, True]

ex01: 2D Array

  • Arquivos de entrega: array2D.py
  • Objetivo: Obter o formato (shape) de um array 2D, realizar um fatiamento (slice) nativo do Python na primeira dimensão (linhas) e retornar o array fatiado com seu novo formato impresso.
P: Qual é a diferença entre o fatiamento de listas nativas do Python e o fatiamento de arrays do NumPy?
O fatiamento nativo em uma lista 2D do Python (lista de listas) só funciona para selecionar linhas (a primeira dimensão), pois o Python enxerga como uma lista unidimensional de objetos. Para fatiar colunas de forma eficiente ou sub-matrizes, precisaríamos usar a sintaxe multidimensional do NumPy (ex: arr[start:end, col_start:col_end]).
P: Como garantimos que a lista fornecida representa uma matriz 2D válida?
Verificamos se cada elemento de family é uma lista e se todas essas sublistas possuem exatamente o mesmo tamanho. Se houver discrepância no tamanho das linhas, lançamos um ValueError para evitar que a conversão para NumPy resulte em um array unidimensional de tipo object com dimensões incorretas.
P: Como lidamos com a validação de tipos inteiros para os índices de fatiamento?
Usamos a checagem explícita type(start) is not int e type(end) is not int para impedir o uso de booleanos (como True ou False), que são subclasses de int em Python e passariam em um teste tradicional com isinstance().

ex02: Load my image

  • Arquivos de entrega: load_image.py
  • Objetivo: Carregar uma imagem usando Pillow (PIL), convertê-la para RGB explicitamente, transformá-la em um array NumPy, imprimir seu formato (shape) e retornar o array contendo os pixels.
P: Por que é necessário converter a imagem explicitamente para "RGB" com `convert("RGB")`?
Imagens podem ter diferentes formatos como RGBA (com canal alfa de transparência), tons de cinza (L) ou paletizadas (P). A conversão explícita para "RGB" garante consistência nos dados de entrada, de forma que o array NumPy resultante sempre possua exatamente 3 canais de cor e o formato (Altura, Largura, 3).
P: Como lidamos com erros de arquivos não encontrados ou formatos inválidos/corrompidos sem causar crash no programa?
Utilizamos blocos try-except capturando exceções específicas do Python e do Pillow como FileNotFoundError e PIL.UnidentifiedImageError. Capturando essas exceções, exibimos uma mensagem de erro clara e retornamos None em vez de permitir que o erro cause o encerramento abrupto (crash) do script.

ex03: Zoom on me

  • Arquivos de entrega: load_image.py, zoom.py
  • Objetivo: Recortar uma área de 400x400 com 1 canal de cor (escala de cinza) de uma imagem carregada e exibi-la com coordenadas nos eixos X e Y.
P: Como o fatiamento no NumPy `img[100:500, 380:780, 0:1]` funciona para fazer o zoom?
Nós fatiamos o array multidimensional especificando os intervalos para cada dimensão: 100:500 para o eixo Y (altura de 400px), 380:780 para o eixo X (largura de 400px) e 0:1 para selecionar apenas o primeiro canal de cor (Red), reduzindo o canal para escala de cinza de canal único. O resultado final possui shape (400, 400, 1).
P: Por que o output exibe `(400, 400, 1) or (400, 400)`?
A primeira tupla representa as dimensões reais do array fatiado mantendo a dimensão de canal (3D com profundidade 1). Ao aplicar sliced.shape[:2], omitimos a última dimensão do canal, obtendo (400, 400), que representa a imagem em formato 2D tradicional.
Exemplo de Output Esperado:
The shape of image is: (768, 1024, 3)
[[[120 110  97]
  [121 111  98]
  ...]]
New shape after slicing: (400, 400, 1) or (400, 400)
[[[167]
  [180]
  ...]]
